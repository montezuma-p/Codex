üîê Secrets Configuration for Complete Automation

This document explains how to configure the necessary secrets for all automation to work perfectly.

üöÄ How to Configure

1. Access Repository Settings

  1. Go to your repository on GitHub
  2. Click on Settings
  3. In the sidebar, click on Secrets and variables ‚Üí Actions

2. Configure PyPI Token (Required for automatic publishing)

2.1. Create Token on PyPI

  1. Access pypi.org
  2. Log into your account
  3. Go to Account settings ‚Üí API tokens
  4. Click Add API token
  5. Set:
  6. Token name: github-actions-codex (or other name)
  7. Scope: Entire account (for simplicity) or Specific project (more secure)
  8. Click Add token
  9. COPY THE TOKEN (only appears once!)

2.2. Add to GitHub

  1. On GitHub, in Secrets and variables ‚Üí Actions
  2. Click New repository secret
  3. Configure:
  4. Name: PYPI_API_TOKEN
  5. Secret: Paste the PyPI token (starts with pypi-)
  6. Click Add secret

3. Configure Environment (Recommended)

For greater security, configure an environment called pypi:

  1. Go to Settings ‚Üí Environments
  2. Click New environment
  3. Name: pypi
  4. Configure:
  5. Required reviewers: Add yourself (optional)
  6. Deployment branches: Only master and main
  7. In Environment secrets, add:
  8. Name: PYPI_API_TOKEN
  9. Value: Your PyPI token

üîß Required Secrets

Secret Description Required
PYPI_API_TOKEN Token to publish to PyPI ‚úÖ Yes
GITHUB_TOKEN Automatic from GitHub ‚úÖ Automatic

üöÄ Making a Release

Two options:

Option 1: Automated Script (Recommended)

make release

Option 2: Manual

# 1. Update version in pyproject.toml
# 2. Commit
git add pyproject.toml
git commit -m "chore: bump version to 1.2.0"

# 3. Create tag
git tag -a "v1.2.0" -m "Release version 1.2.0"

# 4. Push
git push origin master
git push origin v1.2.0

üîç Monitoring

Track workflows at: - GitHub ‚Üí Actions (to see executions) - PyPI ‚Üí Your projects (to see publications) - GitHub Pages (to see documentation)

‚ùó Troubleshooting

Error: "PYPI_API_TOKEN not found"

  • Check if the secret is configured correctly
  • Confirm the name: PYPI_API_TOKEN (case-sensitive)

Error: "Authentication failed"

  • Regenerate the token on PyPI
  • Update the secret on GitHub

Workflow doesn't execute

  • Check if you're on master or main branch
  • Confirm that .github/workflows/*.yml file is committed
